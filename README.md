# buy_tickets_db
Веб-приложение, имитирующее редактирование базы данных билетов админом и выбор и покупку рейса пользователем.
Проект реализует базовую логику работы билетной платформы: хранение данных, управление событиями, бронирование мест и обработку транзакций.

## Обзор функционала
Проект предоставляет следующие ключевые возможности:
  + управление событиями — создание, редактирование и удаление мероприятий;

  + бронирование мест — выбор свободных мест и оформление брони;

  + хранение данных — структурированное хранение информации о событиях, местах и пользователях;

  + история транзакций — отслеживание совершённых бронирований.

### Структура проекта


    buy_tickets_db/

    │   └── scen_auth/        # Модуль аутентификации и авторизации
    ├── templates/            # HTML-шаблоны для веб-интерфейса
    │   ├── end.html          # Страница завершения бронирования
    │   ├── index.html        # Главная страница
    │   └── response.html     # Страница ответа/статуса
    ├── dbcon.py              # Подключение к базе данных
    ├── main.py               # Основной файл приложения
    ├── sql_provider.py       # Работа с SQL-запросами
    └── access.py             # Логика доступа и прав пользователей
    
### Описание ключевых компонентов
  1. Конфигурационные файлы (configs/)

    config.json — основные настройки приложения (порт, хост, параметры БД).

    access.json — права доступа, роли пользователей и ограничения.

  2. Модули приложения (blueprints/)

    basket — управление корзиной бронирования (добавление/удаление мест).
    
    order — обработка заказов, валидация данных, финальное бронирование.
    
    scen_auth — аутентификация пользователей, проверка прав доступа.

  3. Веб-интерфейс (templates/)
    
    index.html — главная страница с перечнем событий.
    
    response.html — отображение статуса операции (успех/ошибка).
    
    end.html — финальная страница после успешного бронирования.

  4. Основные скрипты
     
    main.py — точка входа приложения, инициализация Flask, маршрутизация.
    
    dbcon.py — подключение к БД, управление соединениями.
    
    sql_provider.py — абстракция SQL-запросов, работа с таблицами.
    
    access.py — логика проверки прав доступа, ролевая модель.
